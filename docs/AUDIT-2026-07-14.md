# American BioCarbon - Public Site Audit

## Executive Summary

This audit consolidates findings across brand/copy, accessibility, SEO, performance, and code-consistency dimensions for the American BioCarbon site (`/Users/jaydee3/VEJ SALES DEPT/website/`). The most urgent issues are structural: hash-based routing plus a client-side URL rewrite make every page collapse to a single non-indexable URL, and several conversion-critical CTAs promise one action but link to another. A cluster of unqualified/inconsistent product claims (carbon-neutral vs. carbon-negative, hard 5:1, a fabricated case study) also poses claim-discipline and compliance risk that should be resolved before publish.

## Critical

### C1. Hash-router URLs are not independently indexable
**Location:** `app.js:1242-1283` (router), `index.html:29`
The entire site navigates via hash fragments (`#/product/...`, `#/industry/...`). Crawlers discard everything after `#`, so every route resolves to the site root - no product, industry, compare, or about page can rank as a distinct result. `<main id="app">` is empty in served HTML, so non-JS crawlers see a blank page.
**Fix:** Move to History API `pushState` path routing (the Netlify `_redirects` 200 rewrite already serves `index.html` for any path, so infra supports it), or pre-render/SSG each route. Read `location.pathname` instead of `location.hash` and emit real `<a href="/product/...">` links.

### C2. On-load script rewrites clean SEO URLs back into hash URLs
**Location:** `index.html:43-52`
An inline IIFE takes a clean path like `/industry/animal-bedding` and `history.replaceState`s it to `/#/industry/animal-bedding`, so any indexed clean URL is immediately converted client-side into a non-indexable hash URL - guaranteeing search engines only ever settle on fragments.
**Fix:** Remove the reverse translation; drive the router from `pathname`, keep clean URLs in the address bar, and pair with per-route canonical tags. (Fix alongside C1.)

## High

### H1. Primary sample CTA mislabeled "Request quote"
**Location:** `data.js:47`
`CTA.sample = { label:"Request quote", href:"#/request-sample" }` - used as the primary/secondary action across Home offer, finalCta, and many product/industry pages. Users are told "Request quote" but land on the sample form; the real quote CTA is separate. Conversion-breaking promise/destination mismatch.
**Fix:** Relabel to `"Request a sample"` (or `"Get a sample kit"`), keep the href, and audit every `primary:CTA.sample` usage.

### H2. Fabricated case-study numbers presented as a real result
**Location:** `data.js:1158-1162`
`ENV_REMEDIATION.caseStudy` states specific outcomes as fact (18 vs. 7 truck deliveries, 72 tons of disposal billing saved, "zero compliance callbacks") with no `placeholder` flag, so it renders as a genuine customer case study - exactly the invented measurable-outcome claim the CLAIMS table warns against. Same risk in `ENV_REMEDIATION.whyABC` and `RESELLERS` absolutes.
**Fix:** Remove the numbers or label the block clearly as an illustrative/hypothetical model until a real, attributable case exists; add "validate in your process" qualifiers.

### H3. Absorbents described as both "carbon-neutral" and "carbon-negative"
**Location:** `data.js:306` (and 341, 538 vs. 409, 680, 1186)
The same absorbent product is called "carbon-neutral" in product copy but "carbon-negative" in the Home comparison table, oil-gas fit, and resellers copy. Carbon-negative is a stronger, verifiable-claim-bearing term; the inconsistency is an accuracy and claim-discipline problem.
**Fix:** Decide the defensible position per product (biochar = carbon-negative on H/C basis; absorbents likely carbon-neutral unless deployment-backed) and make every absorbent reference consistent - fix lines 409, 680, 1186 or the product copy.

### H4. Industries animal-bedding copy breaks the stated no-ammonia/odor/cost-savings discipline
**Location:** `data.js:860-867`
The code's own guardrail (line 1048) forbids odor/ammonia/cost-savings claims for bedding, and the deep `ANIMAL_BEDDING` page honors it - but `INDUSTRIES['animal-bedding']` does not: h1 "Drier Litter, Fewer Bags Per House," "fewer bags" usage-savings copy, and explicit "ammonia and odor" language. Two contradictory bedding structures = compliance risk plus voice inconsistency.
**Fix:** Bring the INDUSTRIES entry in line with the guardrail (drop "fewer bags"/cost framing and ammonia/odor language; keep to absorbency/moisture and "measure during trial"), or consolidate to a single bedding content source.

### H5. Focusable CTA links remain tabbable inside `aria-hidden` inactive carousel slides
**Location:** `app.js:242-250` (CSS `styles.css:68`)
Inactive hero slides get `aria-hidden="true"` but still contain real anchor buttons. `pointer-events:none` blocks the mouse but not keyboard focus, so keyboard/SR users tab into invisible controls - WCAG 4.1.2 / 1.3.1 failure.
**Fix:** Add `inert` to inactive slides (or `tabindex="-1"` on their anchors) in both `initHeroCarousel`'s `go()` and the initial render.

### H6. Multiple small-text colors fail 4.5:1 contrast
**Location:** `styles.css:7` (`--mute:#8a8e96` ≈ 2.9:1 on white)
Used pervasively for *informational* copy, not decoration: `.form-note` (consent/legal note), `.pdp-secure`, `.count`, `.shop-doc`, `.doccard .cat`, `.leadnote`, `.buycard .notify`, `.pdp-meta`.
**Fix:** Darken `--mute` to ≈ `#6b7280` or darker (≥4.5:1 on both `#ffffff` and `--paper-2 #eef1f5`); re-verify each usage on the paper-2 background.

### H7. Kicker/eyebrow label color fails contrast on light backgrounds
**Location:** `styles.css:60` (`--gold-d #4d6ba3`, 12.5px uppercase ≈ 3.4:1 on white)
Also affects `.card .icp`, `.steps-list .sn`, `.lane em`, doc/shop link text, and FAQ +/- markers - all meaning-bearing.
**Fix:** Use a darker tone for small label/link text (e.g. `#3a5588` or navy `--graphite #1a376c`), or increase size/weight to qualify as large text. Verify against white and `#eef1f5`.

### H8. Footer address text (red on navy) fails contrast badly

`footer.site .addr` uses `#d7153f` on navy `#1a376c` ≈ 2.6:1 for 13.5px legal name/address text.
**Fix:** Use a light footer color (e.g. `#c3cbdc`, as other footer text) rather than brand red; move the red accent to a non-text element.

### H9. No canonical link tag anywhere

No `<link rel="canonical">` exists and `setMeta` only updates title + description. With hash routing, query-string variants and the root all present as one URL with no canonical signal.
**Fix:** Add a canonical link to the head and have `setMeta` update its href to the clean absolute URL per route (once real paths exist per C1/C2).

### H10. Open Graph / Twitter tags are static, never updated per route

### H11. `twitter:card` is `summary_large_image` but no `og:image`/`twitter:image`

### H12. No structured data (JSON-LD)

### H13. Product bag images are 800-960KB PNGs (~4.3MB for 5)
**Location:** `assets/products/*.png`
PNG is the wrong codec for rendered product shots; `assets/` totals 6.9MB and images dominate page weight/LCP. (JS/CSS are Brotli-compressed by Netlify and minor by comparison.)
**Fix:** Convert bag renders to WebP/AVIF at display size (expect ~85-95% reduction, ~50-120KB each); re-encode the largest industry JPGs (~290KB) to display size at quality ~75; serve via `<picture>`/`.webp` (already used for logo and hands assets).

### H14. Entire page is client-rendered by two non-deferred scripts
**Location:** `index.html:54-55`

**Fix:** At minimum add `defer` to both tags (already dependency-ordered). Better: prerender critical routes or inline above-the-fold markup so LCP doesn't wait on 195KB of JS.

### H15. Undefined CSS variable `var(--brand-crimson)` renders no color
**Location:** `app.js:836`
`renderTechnical()` sets `border-left:4px solid var(--brand-crimson)`, but no `--brand-crimson` token exists (leftover from pre-rebrand palette). With no fallback the whole shorthand is invalid, so the "How it works" callout gets the wrong (currentColor) border.
**Fix:** Replace with the real token (`var(--gold)` = crimson `#d7153f`, or `var(--red)`); grep for other `--brand-*` references.

### H16. Design tokens are semantically wrong after the rebrand - names lie about values
**Location:** `styles.css:5-6`
The rebrand kept old names but swapped values: `--green:#4d6ba3` / `--green-d:#24478a` are blues; `--gold:#d7153f` is crimson; `--gold-d:#4d6ba3` is blue. Every future edit that trusts a token name picks the wrong color - the root cause of contributors hardcoding hex (see M2). (Note: this is *naming* debt; the visual palette enforcement itself was completed this session.)
**Fix:** Rename to real semantics (`--navy`/`--navy-d`, `--crimson`/`--crimson-d`) and update references; at minimum add a comment documenting that `--gold`==crimson and `--green`==navy so nobody "re-fixes" them.
